Model analisis spasial

Skor peta dihitung dari kepadatan kejadian, kedekatan terhadap CCTV, jumlah kejadian sekitar radius 1 km, serta status cakupan kamera. Layer batas administrasi diberi warna seperti choropleth agar pola prioritas per kecamatan langsung terbaca.

Auto-load dari file HTML

Layer dibaca otomatis dari file assets/data/kota_yogyakarta_admin.zip. ZIP asli sudah disederhanakan menjadi satu layer administrasi desa/kelurahan agar stabil dibaca browser. Jika ZIP gagal karena library, dashboard otomatis memakai GeoJSON cadangan.

Auto-load dari CSV

Data kejadian dibaca otomatis dari assets/data/Data Kejahatan Klitih.csv. Kolom yang dipakai: tanggal, bujur, lintang, kecamatan, dan URL sumber.

Network jalan evidence-based

Layer network sekarang tidak lagi menaikkan skor hanya karena garis jalan manual. Skor dihitung dari kejadian klitih yang benar-benar dekat koridor, ketersediaan CCTV di sekitar ruas, dan kepadatan penduduk kecamatan terkait. Ruas tanpa kejadian akan otomatis dibatasi sebagai risiko rendah.

Catatan: network memakai koridor jalan yang ditautkan dengan lokasi CCTV Pemkot dan titik kejadian dari CSV. Ini lebih aman daripada menebak semua jalan, tetapi tetap indikatif karena belum memakai data jaringan jalan resmi bersegmen lengkap.
